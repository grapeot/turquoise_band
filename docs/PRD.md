# PRD：月全食绿松石带的物理重现

## 一句话

从第一性的辐射传输出发，计算月全食期间月面颜色随大气穿透几何的变化，定量重现并最终渲染那条介于暗红本影与正常月光之间的**绿松石带（turquoise band）**。

## 背景与现象

月全食时月亮并不会消失，而是被折射进地球本影的阳光照亮。这束光在到达月面之前，掠过地球大气的边缘（limb），穿过的大气路径极长（切向穿透，气柱可达垂直方向的数十倍）。沿月面径向，从本影中心到边缘，对应的"擦边高度"不同，看到的颜色也不同：

- **本影深处（暗、低擦边高度）**：阳光穿过稠密的低层大气，瑞利散射 ∝ λ⁻⁴ 把蓝光几乎散射殆尽，只剩红光穿透 → 深红/铜红（"血月"）。
- **绿松石带（中等擦边高度，~平流层）**：穿透路径主要落在平流层。这里瑞利散射对蓝端的衰减已经减弱，但臭氧层的 **Chappuis 吸收带（~500–700nm，峰值约 600nm）** 选择性吃掉橙红光，使透射光相对偏向青绿 → 绿松石色窄带。
- **半影/边缘（亮、高擦边高度）**：穿透大气很薄，接近未经强烈选择性衰减的直射阳光 → 接近正常月光的白。

核心物理判断：**绿松石带是臭氧 Chappuis 吸收主导、瑞利散射退居次要的高度窗口的产物。** 红区是瑞利主导，白区是低光程。三个区各有一个 dominant 物理量。

## 目标（成功标准）

按优先级，每一层都是一个可验证的闭环：

1. **L0 — 物理对（必须）**：算出"擦边高度 → 透射光谱 → RGB/色相"的映射。画出色相、亮度随月面径向位置变化的曲线，并自查是否捕捉到 **暗红 → 绿松石 → 白** 的趋势（色相先红、中段转青绿、亮度急升后转白）。matplotlib 曲线即可，低分辨率亦可。这是项目的地基，先到这。
2. **L1 — 定量校核**：绿松石带出现的擦边高度、对应的月面径向位置、带的色相角，与文献/观测照片的定性描述一致（量级对、趋势对）。
3. **L2 — 合成照片（终点）**：把月盘几何接进来，每个月面像素映射到对应的大气穿透路径，渲染出一张 photorealistic 的月食月盘，肉眼可见红心 + 绿松石环 + 白边。

不在范围内（至少 v1）：月面反照率纹理的真实贴图、大气湍流/折射闪烁、多次散射的完整 3D 蒙特卡洛。先做单次散射 + 吸收的视线积分。

## 关键物理量与数据需求

| 物理过程 | 主导区域 | 需要的数据 |
|---|---|---|
| 瑞利散射消光 | 本影深处（红） | 标准大气数密度廓线（US Std Atm 1976）；瑞利散射截面/折射率色散 |
| 臭氧 Chappuis 吸收 | 绿松石带 | 臭氧吸收截面 σ(λ)（500–700nm 实测，如 Serdyuchenko 2014）；臭氧数密度廓线 vs 高度 |
| 入射谱 | 全程 | 大气外太阳辐照谱（ASTM E-490 / Kurucz），或先用 5772K 黑体近似 |
| 几何 | 全程 | 切向穿透的视线几何、地球曲率、本影几何（先解析） |
| 颜色映射 | 全程 | CIE 1931 色匹配函数 → XYZ → sRGB；色相角 |

## 验证哲学（构建闭环）

不追求一开始就 photorealistic。每一步都要有一个能直接看的标量/曲线：透射率谱 → 积分成 XYZ → 色相角与亮度 → 沿径向的曲线。先确认色相曲线的形状对（红→青→白），再逐步提分辨率、换真实数据、上渲染。每替换一个近似为真实数据，对比曲线变化，确认没引入回归。

## 计算与性能

机器：Apple M3 Ultra，32 CPU 核 / 80 GPU 核 / 512GB / Metal(MPS)。
逐波长 × 逐高度 × 逐视线的辐射传输是 embarrassingly parallel。先用 numpy 向量化（波长×路径一次成矩阵），需要时上 MPS（torch）做大网格渲染。把核用满，注意是算法向量化而非 Python 循环。
