# L1 折射几何处方

基于 Mallama (2022) *Lunar Eclipse Phenomena: Modeled and Explained*, arXiv:2112.08966，经第一性原理核对。
（作者是 Anthony Mallama，不是 Robinson；旧版误标，已更正。）

## 核心：擦边高度 → 本影内位置（单一映射）

阳光掠过地球大气 limb 被折射 α(h) 弯入本影。折射角正比于切点局地密度：

```
α(h) = α0 · exp(-h/H),   α0 = 0.0204 rad (70 arcmin),   H = 8 km
```

- **α0=70 arcmin 已含切点两侧对称双段**（日-月擦边光线穿大气路程是地平折射 35' 的两倍），不再 ×2。
- α(h) 消色差——颜色梯度全部来自波长相关的**消光**，折射只负责把不同 h 的光分拣到不同本影半径。

落点（带符号，距本影中心 = 反日轴）：
```
r(h) = (R⊕ + h) − α(h)·d_moon
```
其中 b=R⊕+h 是光线的 impact parameter（离地心最近距离），α(h)·d_moon 是该光线向轴弯进的横移量。
负值 = 越过中心落对侧（极低 h、α 接近 70' 的强折射光线）。这是 Mallama 点源模型的"距影心距离"，
坐标系零点是**反日点轴**。归一化 `r_norm = |r|/R_umbra`。

**坐标系澄清（旧版混乱的根源）**：r(h) 的零点是反日轴，不是本影边界。在这个轴上，
观测者看到的**太阳圆盘本影**（41.2' 边界）落在 r≈4601 km 处；r>4601 km 即进入半影内沿。
高 h 光线（弱折射）r→6355 km，远在 41.2' 边界之外的半影里——这不是 bug，是物理事实：
绿松石带就紧贴并略超出本影几何边界。

本影半径（太阳圆盘会聚影锥）：`R_umbra = R⊕ − (R_sun−R⊕)·d_moon/d_sun ≈ 4601 km = 41.2 arcmin`。
注意这是**太阳圆盘**本影；Mallama 表的"距影心 6420 km@h=50"是**点源**轴坐标，二者参照不同，不可混用。

中高 h 端（绿松石带相关）吻合 <2%；低 h 端误差来自 α0/H 取值（Mallama 隐含 α0·d≈8000 km，

## 关键含义：绿松石带的位置与"窄"从哪来

**映射方向**：低 h → 强折射 → 落本影深处（r_norm→0，红核）；高 h → 弱折射 → 落本影外缘（r_norm→1，青绿/白）。

**绿松石带的 h 区间由独立的辐射传输 debug 给出：青绿色(CIELAB hue≈190°)出现在 h≈12-18 km。**
低于这个区间瑞利还没把蓝端压够、偏红；高于这个区间（h≥20 km）转为蓝紫薰衣草色。
按单一映射，h=12-18km → 距影心 41.5-49.8 arcmin，即**紧贴并略超出 41.2' 本影边界**——位于本影最外缘
到半影内沿。这与文献"turquoise fringe just inside the umbral edge"一致。

**"窄"的来源不是几何陡峭，而是 hue 窗口窄 + 边缘亮度悬崖**：
- dr/dh = 1 + α(h)·d_moon/H，在 h=12km 处约 220 km/km（≈2 arcmin per km of h），在 h=40km 处只有 7.6。
  也就是说几何在低 h 端反而**更展开**（dr/dh 大），不是更挤。所以"带窄"不能归因于"边缘 dr/dh 陡"。
- 真正让带窄的是：(a) 产生青绿 hue 的 h 窗口本身只有约 6 km 宽（12-18km）；
  (b) 越靠本影边缘，总消光呈悬崖式下降（Mallama Table B-1：Pos'n 0.39→0.53 之间蓝光 magnitudes-lost
  从 4 跳到 11），亮度迅速归零，可见的青绿只剩边缘极窄一圈。

采样建议：
- **以 h 为内部采样变量**，在 h=8-25km 密采（绿松石带及其两侧过渡都在这里）。
- 画图横轴用**角距 arcmin**展开（h=12→41.5', h=18→49.8'），并叠加 41.2' 边界参考线，
  让"带紧贴边界外沿"看得见。

## 聚焦因子（亮度，不碰色相）

```
|dh/dr| = 1/(1 + α(h)·d_moon/H)
```
低 h（强折射）→ 因子小 → 光会聚浓缩 → 本影中心更亮（红核虽消光重却最亮的几何原因）。
乘到亮度 Y 上；色相不受影响。r→0 处需软下限正则化（太阳非点源）。

## 之前"矛盾"的真正根因（已废弃的对侧-limb 假说）

曾经的错误诊断认为：贴轴公式把绿松石带(h~20-40km)算到 50-57 arcmin（超出 41.2' 边界），
于是引入第二条"对侧-limb 公式" `r_opp(h)=R_umbra−(α·d−h)` 把它强行拉回 38-41'。**这条公式是错的，已删除。**

**第一性反驳**：月面上某个径向坐标 ρ 的点，问"它被哪条光线照亮"——答案是擦过**同一侧** limb、
满足 r(h)=ρ 的那条光线。一个径向坐标对应一个 h，不存在"贴轴 limb 管红核、对侧 limb 管绿松石带"
这种分工。太阳是个圆盘环，月面每点确实收到绕地球 limb 一整圈的光，但沿**单条径向**做 1D 剖面时，
主导该半径亮度/色相的是 impact parameter b=R⊕+h、横移 α(h)·d_moon 后正好落在该半径的那一族光线，
由单调的 r(h) 唯一确定。所以**只有一套映射** `r(h)=(R⊕+h)−α(h)·d_moon`，它精确复现 Mallama Table 3.1 全程。

**那"超出 41.2' 边界"为什么不是 bug**：因为 r(h) 的零点是反日轴，41.2' 是太阳**圆盘**本影边界。
高 h 弱折射光线落在 41.2' 之外的半影内沿，本来就该如此——绿松石带本就位于本影最外缘到半影内沿
这条窄带。把它"拉回 38-41'"反而破坏了与 Mallama 表的吻合。

**方向小结**：低 h（强折射，α→70'）→ r 小/负 → 本影深处（红核，聚焦增亮）；
高 h（弱折射）→ r→6355 km → 本影外缘/半影内沿（绿松石带）。一套 `shadow_radius_signed_km` 贯穿全程。
